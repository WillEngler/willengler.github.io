_This post is intended for the Pitt Computer Science Club security team, tentatively named Team Loki._

Thanks to everyone who could make it to last night’s meeting. Your ideas and suggestions were a huge help. If you couldn’t make it, that’s fine too!

Based on your suggestions, I’ve assembled our first goals as a group. We’ll start by getting comfortable with our tools. We’ll draw heavily from [Violent Python](http://site.ebrary.com/lib/pitt/detail.action?docID=10623061).

Our First Mission
-----------------

We’ll work on a _big_ group project to get us comfortable with working in Python and collaborating on Github. We’ll work out of chapter 5 of Violent Python to write a packet sniffing script. Take a look at the section titled “Building a Wireless Google Key Logger”. Basically, we can write a script that will show what someone around us is searching for on Google by using Bluetooth.

We can start by re-implementing the Google sniffer. Then, everyone can make their own parsing scripts to interpret a wider range of web requests (e.g. someone writes a parser for Bing). That way, we can all contribute while working on a similar range of programming problems.

Once we have our sniffer working on our laptops, we can be extra devious and run it on a [Raspberry Pi](http://www.raspberrypi.org/). I have one I can contribute to the cause. If we write a program for the Pi that keeps our packet sniffer running continuously, we can stick the tiny Pi into a bookbag or purse and grab packets discreetly from all over Pittsburgh. Yet more programming fun can be had by writing programs that can analyze all of that data and tell us something meaningful about it.

Getting Started with Python
----------------------------

If you’re new to Python (like me!), here are some resources. If you have _just_ started programming and aren’t comfortable in any language yet, Code Academy’s [Python tutorials](http://www.codecademy.com/en/tracks/python) are _very_ user friendly. You just code in the browser. If you already know your way around loops and OOP, there is a [nice site](http://python4java.necaiseweb.org/) that rapidly teaches Python to people who know Java. And then there’s my favorite resource: the [official docs](https://docs.python.org/2/). It has a tutorial that I like.

By the way, I think we should standardize on Python 2.7 instead of 3.4. For the uninitiated, there is a split in the Python community. Version 3 introduced new features but broke backwards compatibility. We’ll be using lots of 3rd party libraries and I’m not confident that they’ve been ported to Python 3. 

You can [download Python](https://www.python.org/download/releases/2.7.8/) and run it on your own machine. However, I prefer to do development in virtualized environments. As such, I’ll try to get a [Vagrantfile](http://willengler.com/devops/2014/08/19/vagrant-intro/) up on Github soon for us to share.

Building a Victim
------------------

It would be  a stretch to call packet sniffing hacking. Eventually, we’ll want to penetrate systems. While we’re working on the packet sniffer, we can prepare sandbox machines that we will mercilessly assault.

Because we’re novice hackers, let’s make a soft target. We’ll use the [Metasploitable](https://community.rapid7.com/docs/DOC-1875)
Linux distribution as our hapless victim. Everyone’s first step should be to install a Metasploitable virtual machine. You can follow the steps in this [helpful tutorial](https://community.rapid7.com/message/4137#4137). Take care that you only configure the VM for a host-only network. Actually exposing it to the internet is begging for trouble.

While we’re all setting up our own private victim machines, I’ll get started on making a “community machine” that we can hack together. I’m going to sacrifice an old Dell laptop that sits in my drawer, unloved. I might keep it in the CS Lounge so that people can stop in and connect to it over ethernet. If the club gets its own project space (we’re trying!) then we can keep it there instead.

After we’ve set up our victims, we can start thinking of how to attack them.
